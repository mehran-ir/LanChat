# LAN Chat — چت و اشتراک فایل تحت شبکه محلی

نرم‌افزاری ساده برای گفتگو و ارسال فایل بین کامپیوترهای متصل به یک شبکه محلی (LAN)، بدون نیاز به سرور مرکزی یا اینترنت. هر کامپیوتر با **نام کامپیوتر** (Hostname) خودش در برنامه شناسایی می‌شود و برنامه می‌تواند به‌صورت خودکار کامپیوترهای دیگری که همین برنامه را اجرا کرده‌اند در شبکه پیدا کند.

## امکانات
- کشف خودکار کامپیوترهای دیگر در شبکه (دکمه «اسکن شبکه»)
- افزودن دستی یک کامپیوتر با وارد کردن IP آن
- ارسال و دریافت پیام متنی آنی
- ارسال و دریافت فایل با هر حجم (فایل‌های دریافتی در پوشه `received_files` کنار برنامه ذخیره می‌شوند)
- بدون نیاز به سرور مرکزی — هر کپی از برنامه هم کلاینت است هم سرور

## نحوه ساخت فایل اجرایی (exe) — بدون نیاز به نصب پایتون (روش ابری/رایگان)

اگر هیچ‌کدام از کامپیوترهای شما پایتون ندارد، می‌توانید بگذارید یک سرور ویندوزی **رایگان در فضای ابری GitHub** فایل exe را برایتان بسازد. این پروژه از قبل شامل فایل تنظیمات لازم (`.github/workflows/build.yml`) است.

مراحل (فقط نیاز به یک حساب رایگان GitHub دارید، نه نصب چیزی روی کامپیوتر خودتان):

1. وارد سایت [github.com](https://github.com) شوید و اگر حساب ندارید، یک حساب رایگان بسازید.
2. روی «+» بالای صفحه کلیک کنید → **New repository** → یک نام دلخواه (مثلاً `lanchat`) بدهید → **Create repository**.
3. در صفحه‌ی مخزن تازه ساخته‌شده، روی لینک **uploading an existing file** کلیک کنید.
4. تمام فایل‌ها و پوشه‌های داخل این بسته (شامل پوشه‌ی `.github`) را با کشیدن و رها کردن (drag & drop) آپلود کنید، سپس **Commit changes** را بزنید.
   - نکته: پوشه‌ی مخفی `.github` باید حتماً با همان ساختار آپلود شود؛ اگر مرورگر شما پوشه‌ها را در drag&drop قبول نمی‌کند، از گزینه‌ی **Add file → Upload files** و انتخاب همه فایل‌ها به‌صورت یک‌جا (به همراه ساختار پوشه در فایل مرورگر) استفاده کنید.
5. به تب **Actions** در بالای صفحه‌ی مخزن بروید. یک اجرای خودکار (workflow) به نام «Build Windows EXE» شروع می‌شود (چند دقیقه طول می‌کشد).
6. وقتی علامت ✅ سبز کنار آن ظاهر شد، روی آن کلیک کنید، پایین صفحه بخش **Artifacts** را پیدا کنید و فایل `LANChat-windows-exe` را دانلود کنید. داخل آن zip، فایل `LANChat.exe` واقعی و آماده اجرا روی ویندوز است.
7. همان فایل exe را روی هر تعداد کامپیوتر ویندوزی که می‌خواهید کپی و اجرا کنید — دیگر نیازی به هیچ نصبی نیست.

اگر اجرا نشد، از تب Actions روی workflow کلیک کنید و دکمه‌ی **Run workflow** را بزنید تا دستی اجرا شود.

## روش جایگزین: ساخت exe روی یک کامپیوتر ویندوزی با پایتون

اگر دسترسی موقت به یک کامپیوتر ویندوزی (حتی کامپیوتر یک دوست یا همکار) دارید که می‌توانید روی آن پایتون نصب کنید:

1. تمام فایل‌های این پروژه را در یک پوشه کپی کنید.
2. فایل `build.bat` را اجرا کنید (دابل‌کلیک).
3. پس از پایان، فایل نهایی در مسیر `dist\LANChat.exe` قرار می‌گیرد.
4. همین یک فایل `LANChat.exe` را کپی کرده و روی هر کامپیوتر ویندوزی دیگری که می‌خواهید (نیازی به نصب پایتون روی آن‌ها نیست) قرار دهید و اجرا کنید.

> اگر پایتون ندارید، از python.org نسخه 3.9 به بالا را نصب کنید و هنگام نصب تیک «Add python.exe to PATH» را بزنید.

### اجرای مستقیم بدون ساخت exe (برای تست)
```
python main.py
```

## نحوه استفاده
1. برنامه را روی هر کامپیوتری که می‌خواهید در گفتگو شرکت کند اجرا کنید.
2. در بالای پنجره، نام و IP همان کامپیوتر نمایش داده می‌شود.
3. روی دکمه **«اسکن شبکه»** کلیک کنید تا کامپیوترهای دیگری که برنامه را اجرا کرده‌اند به‌صورت خودکار در لیست سمت چپ ظاهر شوند.
4. اگر کامپیوتری به‌صورت خودکار پیدا نشد (مثلاً به دلیل تنظیمات شبکه)، از دکمه **«افزودن دستی»** استفاده کرده و IP آن را وارد کنید.
5. روی نام یک کامپیوتر در لیست کلیک کنید تا پنجره گفتگوی آن باز شود.
6. پیام خود را تایپ و Enter بزنید، یا روی **«ارسال فایل»** کلیک کنید تا فایل انتخاب و ارسال شود.
7. فایل‌های دریافتی به‌صورت خودکار ذخیره می‌شوند؛ با دکمه **«باز کردن پوشه فایل‌های دریافتی»** می‌توانید آن‌ها را ببینید.

## نکات مهم درباره شبکه و فایروال
- برنامه از پورت **UDP 54545** برای شناسایی کامپیوترها و پورت **TCP 54546** برای ارسال پیام/فایل استفاده می‌کند.
- در اولین اجرا، **فایروال ویندوز (Windows Defender Firewall)** ممکن است پیام اجازه دسترسی نمایش دهد؛ گزینه **«Allow access»** را برای شبکه خصوصی (Private) تایید کنید تا برنامه به درستی کار کند.
- همه کامپیوترها باید در یک زیرشبکه (مثلاً همه Wi-Fi یا LAN یک روتر/سوییچ) باشند تا Broadcast کشف خودکار به آن‌ها برسد. در شبکه‌های بزرگ‌تر سازمانی که Broadcast مسدود شده، از گزینه «افزودن دستی» با وارد کردن IP مقصد استفاده کنید.
- می‌توانید IP کامپیوتر مقصد را با دستور `ipconfig` در Command Prompt آن کامپیوتر پیدا کنید.

## ساختار پروژه
```
lanchat/
├── main.py          نقطه ورود برنامه
├── gui.py           رابط گرافیکی (Tkinter)
├── discovery.py      کشف کامپیوترها (UDP Broadcast)
├── server.py         سرور TCP دریافت پیام/فایل
├── client.py         ارسال پیام/فایل به مقصد
├── netprotocol.py     پروتکل فریم‌بندی پیام/فایل
├── utils.py          توابع کمکی (نام کامپیوتر، IP محلی)
├── build.bat         اسکریپت ساخت exe با PyInstaller
└── requirements.txt
```

## محدودیت‌های شناخته‌شده
- کشف خودکار مبتنی بر UDP Broadcast است و معمولاً بین VLAN های مختلف یا شبکه‌های Wi-Fi با «AP/Client Isolation» فعال کار نمی‌کند؛ در این موارد از افزودن دستی استفاده کنید.
- تاریخچه گفتگو فقط در حافظه برنامه نگه‌داری می‌شود و با بستن برنامه پاک می‌شود (ذخیره‌سازی دائمی در این نسخه پیاده‌سازی نشده است).
